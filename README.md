# 📊 Dashflow | Sales Analytics

AI-powered sales dashboard designed for ease of use and security.

## 🚀 Connecting to Supabase (Optional)

The backend API is configured to read data from a local CSV by default, but is fully ready to connect to Supabase whenever you are ready.

### 1. Create a Project
Go to [Supabase](https://supabase.com) and create a new project.

### 2. Create the Table
In your Supabase dashboard, go to the **SQL Editor** and run this snippet to create your table:

```sql
CREATE TABLE sales_data (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  date DATE NOT NULL,
  product TEXT NOT NULL,
  channel TEXT NOT NULL,
  orders INTEGER DEFAULT 0,
  revenue DECIMAL(12,2) DEFAULT 0,
  cost DECIMAL(12,2) DEFAULT 0,
  visitors INTEGER DEFAULT 0,
  customers INTEGER DEFAULT 0
);
```

### 3. Import Your Data
Go to the **Table Editor** for your new `sales_data` table and use the **"Import data via CSV"** button to upload your `data/sales_data.csv` file.

### 4. Get Your Credentials
Go to **Project Settings -> API** to copy your:
*   Project URL
*   service_role key (or Anon Key)

### 5. Update Your Settings
Open your local `.env` file and update your credentials while setting the `DATA_SOURCE` to `supabase`:

```env
DATA_SOURCE=supabase
SUPABASE_URL=your_project_url_here
SUPABASE_ANON_KEY=your_anon_key_here
```

### 6. Start the Dashboard
Restart your development server. The API will now serve data directly from Supabase!

```bash
npm run dev
```

## 🛠️ Tech Stack
*   **Frontend**: React + Tailwind CSS
*   **Backend**: Node.js + Express (Secure API Server)
*   **AI**: Google Gemini (Direct Server-to-Server)
*   **Storage**: CSV (Default) or Supabase (Enterprise)
